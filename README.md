# Audio Transcript App (Gemini AI + Next.js)

A full-stack, glassmorphism-themed Next.js application that allows an authenticated admin to upload short audio files and get high-quality transcripts generated by Google's Gemini AI.

## 🚀 Tech Stack
- **Framework**: Next.js (App Router)
- **Database**: PostgreSQL (via Neon)
- **ORM**: Prisma
- **Auth**: Better Auth (Email & Password)
- **AI Engine**: Google Gemini 2.5 Flash (`@google/generative-ai`)
- **Styling**: Vanilla CSS with a premium dark-mode glassmorphism system

## ✨ Features
- **Secure Admin Dashboard**: Only authenticated users can upload and view transcripts.
- **Micro-Animations & Premium UI**: Glow effects, transparent panels, and sleek responsive design.
- **Server Actions**: Audio parsing and AI querying securely handled Server-Side.
- **Intelligent Error Handling**: Built-in size limitations and intuitive API constraints.

## 📦 Setup Instructions

**1. Clone the repository**
```bash
git clone https://github.com/dheerajsingh9334/ai-mp3-transcript.git
cd ai-mp3-transcript
```

**2. Install Dependencies**
```bash
npm install
```

**3. Configure Environment Variables**
Create a `.env` file in the root of your project:
```env
DATABASE_URL="postgresql://your_db_username:your_db_password@your_host/your_db?sslmode=require"
AUTH_SECRET="your_highly_secure_random_string"
AUTH_URL="http://localhost:3000" # Change this for production!
GEMINI_API_KEY="your_google_gemini_api_key_here"
```

**4. Generate Database Schema**
```bash
npx prisma generate
npx prisma db push
```

**5. Start the Development Server**
```bash
npm run dev
```

Visit the application at `http://localhost:3000`. 
*(Note: To seed the admin user, visit `/api/seed` in your browser once while the server is running.)*

## 🌐 Deployment (Railway)
This application is fully optimized for Railway.
1. Deploy this repository to a new Railway project.
2. In Railway, configure the exact environment variables from Step 3.
3. Use the Railway "Generate Domain" feature to get a public URL.
4. Set the `AUTH_URL` variable to your new public Railway domain.
5. Redeploy the project!
